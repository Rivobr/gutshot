# GUTSHOT Poker Club
## Project Architecture

Версия: MVP 1.0

---

# Описание проекта

GUTSHOT Poker Club — это цифровая система управления офлайн-покерным клубом, полностью интегрированная с Telegram.

Проект предназначен для автоматизации работы клуба:

- регистрация игроков;
- управление турнирами;
- QR Check-In;
- рейтинг игроков;
- система XP;
- статистика;
- Telegram-уведомления;
- административная панель.

Проект не является онлайн-покером и не предоставляет возможность играть на деньги через интернет.

# Основные компоненты

## Telegram Bot

Отвечает за:

- авторизацию;
- отправку уведомлений;
- запуск Mini App;
- взаимодействие с пользователями.

---

## Mini App

Основное приложение игрока.

Функции:

- Главная
- Турниры
- Регистрация
- Мой турнир
- QR
- Рейтинг
- Профиль
- История

---

## Backend

NestJS приложение.

Содержит всю бизнес-логику.

Backend является единственным источником истины.

Никакая логика не должна находиться во Frontend.

---

## PostgreSQL

Основная база данных проекта.

Хранит:

- пользователей;
- турниры;
- регистрации;
- XP;
- QR;
- уведомления;
- администраторов.

---

## Redis

Используется для:

- кеширования;
- хранения временных QR;
- хранения JWT blacklist (при необходимости);
- временных данных.

---

## Admin Panel

Внутренняя CRM клуба.

Доступна только администраторам.

Позволяет:

- создавать турниры;
- управлять регистрациями;
- сканировать QR;
- завершать турниры;
- просматривать статистику.

# Архитектурные принципы

- Backend содержит всю бизнес-логику.
- Frontend отвечает только за отображение данных.
- Все изменения проходят через Backend API.
- Prisma — единственный способ работы с базой данных.
- Один модуль — одна ответственность.
- Вся система строится вокруг сущности Registration.
- Уровень игрока вычисляется по количеству XP.
- Статистика рассчитывается автоматически.
- Telegram является единственным способом авторизации игроков.
